[퍼플렉시티](https://www.perplexity.ai/) API는 사용자가 직접 충전한 크레딧을 기반으로 호출 횟수와 토큰 사용량에 따라 비용을 지불하는 선불제 구조로 운영됩니다.

## **전체 내용 요약**

[Perplexity API](https://www.perplexity.ai/api-platform)는 사용자가 필요한 만큼 크레딧을 미리 충전하여 사용하는 선불 결제 방식을 채택하고 있습니다.

Pro 구독자의 경우 매달 5달러의 API 크레딧이 제공되며, 검색 전용 API와 Sonar 계열의 언어 모델 API로 구분되어 과금이 이루어집니다.

## **핵심 체크리스트**

- API 사용 전 결제 카드 등록 및 크레딧 선충전 필요 여부 확인하기
- Pro 구독자라면 매월 1일 지급되는 5달러 크레딧 활용하기
- 단순 웹 검색 결과만 필요할 경우 Search API 사용 고려하기
- Sonar 모델 사용 시 컨텍스트 크기(Low/Medium/High)에 따른 비용 차이 인지하기
- 자동 충전(Auto top-up) 설정을 통해 API 중단 방지하기
- 누적 지출액에 따른 티어 상승 및 레이트 리밋 완화 조건 확인하기
## **퍼플렉시티 API 서비스는 무엇인가요?**

최근 다양한 검색 중심의 서비스를 개발하려는 움직임이 많아지면서 관련 API 비용에 대한 관심도 높아지고 있습니다. Perplexity API는 실시간 웹 검색 기능이 결합된 모델을 제공하는데요.

이번 글에서는 퍼플렉시티 api를 사용하는 과정에서 꼭 알아야 할 서비스의 비용 구조와 크레딧 정책을 자세히 알아보겠습니다.

## **선충전 방식의 크레딧 결제 구조**

Perplexity API 서비스는 사용자가 크레딧을 미리 구매하여 잔액을 채워두는 선불 방식을 기본으로 운영됩니다.

계정에 등록된 카드는 향후 결제를 위한 수단이고, 실제 사용량에 맞춰 충전된 크레딧이 차감되는 형태를 띠고 있습니다.

크레딧 잔액이 일정 기준 이하로 떨어질 경우 API 호출이 제한될 수 있으므로 운영 중 주의가 필요하다고 생각합니다.

이러한 중단을 방지하기 위해 잔액이 부족할 때마다 자동으로 결제가 이루어지는 자동 충전 기능을 설정할 수 있습니다.

Pro 구독자는 매달 1일 5달러 상당의 크레딧을 지원받으므로 초기 테스트 시 유용하게 활용할 수 있습니다.

요약포인트: 크레딧 선충전 방식을 통해 예산을 관리하며 Pro 구독자는 매월 무료 크레딧을 제공받습니다.

## **모델별 과금 체계 비교**

| 항목 | 내용 | 비고 |
| --- | --- | --- |
| Search API | $5.00 / 1K requests | 토큰 비용 없음 |
| Sonar 입력 비용 | $1 / 1M tokens | 100만 토큰당 가격 |
| Sonar Pro 입력 비용 | $3 / 1M tokens | 100만 토큰당 가격 |
| Sonar Reasoning 입력 비용 | $1 / 1M tokens | 100만 토큰당 가격 |
| Sonar Deep Research 입력 | $2 / 1M tokens | 검색 쿼리당 별도 비용 발생 |

요약포인트: Search API는 요청 수로만 과금되며 Sonar 계열 모델은 토큰 비용과 요청 비용이 병행됩니다.

## **모델별 요청 비용 및 Pro Search 요금**

Sonar 계열의 툴은 단순히 토큰 양뿐만 아니라 요청 시 포함되는 컨텍스트의 크기에 따라 추가 비용이 발생합니다.

예를 들어 Sonar 기본 모델의 경우 컨텍스트가 Low 단계일 때는 1,000회 요청당 5달러가 부과되지만 High 단계에서는 12달러까지 증가합니다.

에이전틱 검색 기능이 강화된 Pro Search 모드를 활성화하면 기본 모델보다 높은 요청 비용이 적용될 수 있습니다.

이러한 복합적인 과금 구조로 인해 실제 발생 비용을 예측하기 위해서는 평균적인 토큰 소모량과 검색 빈도를 함께 계산해야 합니다.

또한 누적 지출액에 따라 사용량 티어가 올라가며 이를 통해 레이트 리밋 완화 등의 혜택을 기대할 수 있습니다.

- 개발 전 예상되는 요청 횟수와 토큰 사용량 파악하기
- 모델별 입력 및 출력 단가 차이를 확인하여 예산 수립하기
- 비용 효율성을 위해 실시간 검색이 불필요한 경우 일반 LLM 활용 고려하기
- 사용량 티어 상승에 따른 레이트 리밋 변화 모니터링하기
요약포인트: 요청 비용은 컨텍스트 크기에 비례하며 Pro Search 모드는 더 높은 비용이 책정됩니다.

## **Q&A**

Q. Pro 구독자가 받은 5달러 크레딧이 다음 달로 이월되나요?

A. 공식 문서상에는 매월 1일 지급된다는 사실만 명시되어 있으며 이월 여부에 대해서는 구체적인 언급이 없습니다. 따라서 매달 소진하는 방향으로 계획을 세우는 것이 안전하다고 생각합니다.

Q. 크레딧을 실수로 너무 많이 충전했는데 환불이 가능한가요?

A. 충전 후 14일 이내이면서 사용하지 않은 크레딧에 한해 환불 신청이 가능합니다. 이미 사용된 크레딧에 대해서는 환불이 어렵다는 점을 유의해야 합니다.

Q. API 사용량이 많아지면 할인을 받을 수 있나요?

A. 개별적인 가격 할인보다는 누적 지출액에 따른 티어 상승을 통해 레이트 리밋이 완화되거나 베타 기능에 접근할 수 있는 혜택이 주어집니다. 이를 통해 더 많은 데이터를 처리할 수 있는 환경이 조성됩니다.

Q. Search API와 Sonar 모델의 가장 큰 비용 차이는 무엇인가요?

A. Search API는 토큰 비용 없이 오직 요청 횟수로만 비용을 산정합니다. 반면 Sonar 모델은 요청 횟수와 토큰 사용량이 결합되어 최종 비용이 결정되는 차이가 있습니다.

Q. 무료 사용자도 API를 이용할 수 있나요?

A. 무료 사용자도 크레딧을 직접 충전하면 API 서비스를 이용할 수 있습니다. 다만 Pro 구독자에게 제공되는 월간 5달러 크레딧 혜택은 받을 수 없습니다.
